<objective>
Introduce a per-request ArenaAllocator in runAnalysis() so intermediate pass allocations are bulk-freed after each analysis request, preventing unbounded memory growth during long editing sessions.

Purpose: LSP-01 — the current implementation leaks all intermediate allocations (reporter, mod_resolver, declaration collectors, type resolvers, ownership/borrow/thread checkers, propagation checker, topological order slice) into the long-lived allocator. Over hundreds of analysis cycles, this accumulates significant memory.
Output: Updated runAnalysis() that uses a scratch arena for all pass objects while preserving diagnostics and symbols in the long-lived allocator, plus a unit test verifying returned data survives arena deinitialization.
</objective>

Add a unit test after the existing test blocks (after line ~3156) that exercises runAnalysis with `std.testing.allocator`. In Zig debug mode, `std.testing.allocator` wraps GeneralPurposeAllocator which detects:
- Use-after-free (accessing arena memory after deinit)
- Memory leaks (allocations not freed)
- Double-free

The test calls `runAnalysis(std.testing.allocator, ".")` which will likely return an empty AnalysisResult (no valid project in test CWD) but exercises the full arena init/deinit path. Then it calls `freeDiagnostics` and `freeSymbols` on the result. If any returned slice points into freed arena memory, `std.testing.allocator` will detect it.

```zig
test "runAnalysis arena does not leak or corrupt returned data" {
    // std.testing.allocator detects use-after-free and leaks in debug mode.
    // runAnalysis creates a scratch arena internally; returned diagnostics/symbols
    // must be allocated with the long-lived allocator (std.testing.allocator here)
    // and survive the arena deinit.
    const result = runAnalysis(std.testing.allocator, ".") catch |err| {
        // Analysis may fail on missing project structure — that's OK.
        // The important thing is that the arena was cleaned up without error.
        _ = err;
        return;
    };
    // If we got results, verify they can be read and freed without allocator errors.
    // std.testing.allocator will panic on use-after-free or double-free.
    freeDiagnostics(std.testing.allocator, result.diagnostics);
    freeSymbols(std.testing.allocator, result.symbols);
}

test "runAnalysis can be called twice without accumulation" {
    // Two sequential calls prove the arena from the first call is fully released.
    // std.testing.allocator detects leaks at test end.
    for (0..2) |_| {
        const result = runAnalysis(std.testing.allocator, ".") catch |_| {
            continue;
        };
        freeDiagnostics(std.testing.allocator, result.diagnostics);
        freeSymbols(std.testing.allocator, result.symbols);
    }
}
```

Note: `runAnalysis` changes the CWD internally (with defer restore), so the test CWD is preserved. The function handles missing project directories gracefully by returning an empty AnalysisResult, so these tests are safe to run in the test environment.
