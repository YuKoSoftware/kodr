1. **src/templates/example/error_handling.orh** — Add a `throw` section after the existing error handling section (~line 20, after `divide_or_default`). Add:

```
// ---- Throw Statement (error propagation) ----

// throw propagates the error and narrows the type
func divide_with_throw(a: i32, b: i32) (Error | i32) {
    var result = safe_divide(a, b)
    throw result
    return result
}
```

The key here: `result` starts as `(Error | i32)`. After `throw result`, `result` is narrowed to `i32`, so `return result` works without `.value`. The function itself returns `(Error | i32)` so the early error return is valid.

Add a test in the existing test section:
```
test "throw propagation" {
    assert(divide_with_throw(10, 2) == 5)
}
```
