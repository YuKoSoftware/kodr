    ```orhon
    module fail_enum_value

    // Tagged union variant cannot have explicit discriminant
    enum(u8) Bad {
        Foo(i32) = 4
    }

    pub func main() void {
        return
    }
    ```
    This tests D-03: tagged union variant with both fields `(i32)` and explicit value `= 4` is a parse error. The grammar's ordered choice prevents this — `(` takes the field branch, then `= 4` is unexpected. The `pub func main()` is needed to avoid "no entry point" errors.
    Note: Because the grammar uses ordered choice `('=' int_literal / '(' _ param_list _ ')')`, writing `Foo(i32) = 4` means the parser takes the `(` branch for fields, then `= 4` is not part of the grammar rule and becomes a parse error.
