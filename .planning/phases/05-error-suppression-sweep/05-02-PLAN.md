Fix all 6 `catch {}` in collections.zig per D-06. These are data-loss bugs — the caller thinks the item was added but it was silently dropped on OOM.

**Pattern: `catch return` for mutation methods (add/put).** The function returns void, so `catch return` exits immediately. The caller does not get confirmation the item was added, but at least we stop processing rather than silently continuing. Update the comment to explain.

**Pattern: `catch break` for iteration builders (keys/values/items).** These build a result array in a while loop. On OOM, stop collecting rather than silently skipping items. This returns a partial-but-honest result.

Specific changes:

1. **Line 26** `self.inner.append(self.alloc, item) catch {};` in `List.add`:
   Change to: `self.inner.append(self.alloc, item) catch return; // OOM: cannot add item`

2. **Line 80** `self.inner.put(self.alloc, key, value) catch {};` in `Map.put`:
   Change to: `self.inner.put(self.alloc, key, value) catch return; // OOM: cannot put entry`

3. **Line 103** `result.append(self.alloc, entry.key_ptr.*) catch {};` in `Map.keys`:
   Change to: `result.append(self.alloc, entry.key_ptr.*) catch break; // OOM: return partial keys`

4. **Line 112** `result.append(self.alloc, entry.value_ptr.*) catch {};` in `Map.values`:
   Change to: `result.append(self.alloc, entry.value_ptr.*) catch break; // OOM: return partial values`

5. **Line 139** `self.inner.put(self.alloc, item, {}) catch {};` in `Set.add`:
   Change to: `self.inner.put(self.alloc, item, {}) catch return; // OOM: cannot add item`

6. **Line 158** `result.append(self.alloc, entry.key_ptr.*) catch {};` in `Set.items`:
   Change to: `result.append(self.alloc, entry.key_ptr.*) catch break; // OOM: return partial items`
