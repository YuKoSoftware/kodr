    - A STRING_LITERAL "hello @{name}!" returns an .interpolated_string node with 3 parts: literal "hello ", expr identifier "name", literal "!"
    - A STRING_LITERAL "@{x}" returns an .interpolated_string with 1 part: expr identifier "x"
    - A STRING_LITERAL "prefix @{a} middle @{b} suffix" returns 5 parts alternating literal/expr
    - An empty string "" returns a plain .string_literal node
    - Unclosed @{ (e.g. "broken @{name") treats remainder as literal (silent degradation)
  </behavior>
  <action>
Per D-03, D-04, D-05: Replace the single-line `buildStringLiteral` function in src/peg/builder.zig (line 1043-1045) with interpolation-aware logic. Builder-level post-processing per D-06 and research recommendation (no grammar change needed).

Implementation:
1. Get raw token text via `tokenText(ctx, cap.start_pos)` — includes outer double-quotes
2. Guard: if `raw.len < 2`, return plain `.string_literal` (malformed token)
3. Extract inner text: `raw[1..raw.len - 1]`
4. Fast path: if `std.mem.indexOf(u8, inner, "@{") == null`, return plain `.string_literal` (unchanged behavior)
5. Slow path — scan `inner` to build `std.ArrayListUnmanaged(parser.InterpolatedPart)`:
   - When `@{` found: emit preceding text as `.literal` part (arena-duped), find matching `}`, create `.identifier` node from text between `@{` and `}` (arena-duped), emit as `.expr` part
   - When no more `@{`: emit remainder as `.literal` part
   - If no closing `}` found after `@{`: treat remainder (including the `@{`) as literal, break
6. Return `ctx.newNodeAt(.{ .interpolated_string = .{ .parts = try parts.toOwnedSlice(ctx.alloc()) } }, cap.start_pos)`

Key details:
- Use `parser.InterpolatedPart` (NOT InterpolationPart — pitfall 6 from research)
- Arena-dupe all literal slices via `ctx.alloc().dupe(u8, slice)` (pitfall 2)
- Arena-dupe expr_text before passing to newNodeAt (token text lifetime)
- Use `std.mem.indexOfScalarPos(u8, inner, expr_start, '}')` to find closing brace
- Only simple identifier expressions supported (covers all current usage: @{name}, @{x})
