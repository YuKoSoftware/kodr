<objective>
Make bridge .zig sidecar files compile as named Zig modules in the build graph, eliminating file-path @import and cross-module "file exists in two modules" errors.

Purpose: Bridge modules that import types from other bridge modules currently cause Zig build failures because file-path @import creates duplicate module entries. Named modules in build.zig resolve this.

Output: Modified zig_runner.zig (build.zig generation), codegen.zig (import path), and main.zig (data collection) that produce correct named-module build graphs.
</objective>

**New structure of generated build.zig:**

1. **Preamble** (unchanged): std import, build function, target/optimize, str_mod/coll_mod creation

2. **NEW: Bridge module creation** — After the preamble, before lib/exe targets:
   - Iterate all targets. For each target where `has_bridges == true`:
     ```zig
     // Emit:
     const bridge_{module_name} = b.createModule(.{
         .root_source_file = b.path("{module_name}_bridge.zig"),
         .target = target,
         .optimize = optimize,
     });
     ```
   - Then wire bridge-to-bridge imports: if target A has_bridges and imports target B which also has_bridges, emit:
     ```zig
     bridge_{A}.addImport("{B}_bridge", bridge_{B});
     ```
     To determine bridge-to-bridge deps: for each bridge target, check its `lib_imports` — if any imported module also has_bridges, that's a bridge-to-bridge dep.
   - Apply #linkC to bridge modules (not to lib/exe targets). For each bridge target with link_libs:
     ```zig
     bridge_{module_name}.linkSystemLibrary("{lib}", .{});
     bridge_{module_name}.linkLibC();
     ```

3. **Modified lib target emission** (Pass 1):
   - After creating the lib target and adding str_mod/coll_mod imports, also add:
     ```zig
     lib_{name}.root_module.addImport("{name}_bridge", bridge_{name});
     ```
     (only if the target has_bridges)
   - Remove any linkSystemLibrary/linkLibC from lib targets (it now goes on bridge modules)
   - Keep existing lib-to-lib addImport logic

4. **Modified exe target emission** (Pass 2):
   - After creating exe target and adding str_mod/coll_mod imports, for each lib_import that has_bridges:
     ```zig
     exe_{name}.root_module.addImport("{dep}_bridge", bridge_{dep});
     ```
   - Also if the exe itself has_bridges:
     ```zig
     exe_{name}.root_module.addImport("{name}_bridge", bridge_{name});
     ```
   - Do NOT apply linkSystemLibrary/linkLibC to exe targets

5. **Test step** (unchanged logic, but if the test target's module has_bridges, add bridge import)

Before the lib/exe emission passes, build a `StringHashMap(bool)` mapping module_name -> has_bridges for quick lookup. Also build module_name -> link_libs mapping.

For bridge-to-bridge wiring, need to know which bridges import which other bridges. Use the existing `lib_imports` field: if target A's lib_imports contains module B, and B has_bridges, then bridge_A needs addImport for bridge_B.
