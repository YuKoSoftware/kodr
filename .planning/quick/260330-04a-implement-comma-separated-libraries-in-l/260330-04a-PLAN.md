  <behavior>
    - Test 1: "SDL3" (no comma) produces a single link lib "SDL3"
    - Test 2: "vulkan, SDL3" produces two link libs "vulkan" and "SDL3"
    - Test 3: "vulkan,SDL3" (no spaces) produces "vulkan" and "SDL3"
    - Test 4: " vulkan , SDL3 " (extra whitespace) produces "vulkan" and "SDL3"
    - Test 5: Empty segments from ",," are skipped
  </behavior>
  <action>
1. Add a private helper function `splitLibNames` near the top of the file (or near the collectCimport function):
   ```
   fn splitLibNames(name: []const u8) SplitIterator
   ```
   Use `std.mem.splitScalar(u8, name, ',')` and trim each segment with `std.mem.trim(u8, segment, " \t")`. Skip empty segments after trimming.

   Since Zig's split iterator is lazy, the simplest approach is to NOT create a separate function but instead inline the split+trim loop at each of the 3 sites where `lib_name` is currently used.

2. Site 1 (multi-target collectCimport, ~line 608-625): After extracting `lib_name` from the raw string literal, instead of using it directly:
   - Split `lib_name` on ','
   - For each trimmed non-empty segment: do the duplicate check against `registry`, put into registry, and append to `link_libs`
   - The `c_includes` and `c_sources` handling stays OUTSIDE the split loop (they apply once per #cimport, not per lib)

3. Site 2 (single-target root module, ~line 775-780): Same split+trim pattern — split `lib_name` on comma, append each trimmed segment to `link_libs`

4. Site 3 (single-target non-root modules, ~line 808-817): Same split+trim pattern — split on comma, check duplicates per segment, append each

Key points:
- The `name:` string in the #cimport metadata node is a SINGLE string_literal — the comma is INSIDE the string, not in the grammar. No PEG changes needed.
- The cimport_registry key should be the FIRST lib name (or the full original string) for duplicate #cimport detection (D-08). Actually, the registry purpose is to detect duplicate #cimport blocks per library — so each individual split name should be registered separately.
- The `c_includes` and `c_sources` are per-cimport-block, not per-lib — do NOT duplicate them per comma-segment.
  </action>
